Memory and Numbers

The memory in a computer is essentially a large collection of bytes. Every byte has a unique address assigned to it.

Many things are too large to fit into one byte, so bytes in a memory are often categorized into chunks.

  • The address of a chunk is the address of the first byte.

Only 4 byte integers are used in 213.

Big & Little Endian

4 byte integers are stored differently in different processors. Most processors are said to be big-endian, while modern intel processors are said to be little-endian.

Consider the integer 0x12345678. This integer is stored differently based on the type of processor (as shown on the right).

Alignment

Technically items in memory can be placed anywhere. However it is more efficient if memory is aligned.

The CPU accesses memory in different blocks. If memory is unaligned, it is possible that the CPU will have to access more than one block to access the same item in memory.

The figure on the right shows the difference more clearly by using a block of memory that contains a char and an int.

Bit Shifting

Shifting bits is the same as multiplying/dividing by 2.

  • Shifting left by b is the same as multiplying by 2b
  • Shifting right by b is the same as dividing by 2b

Extending/Trunacting Integers

Extending an integer is when you increase the number of bytes used to store the integer.

byte b = -6;
int i = (byte) b; // prints 0xfffffffa

There are two types:

  • Signed extension — used with signed numbers (copies f to the empty bits).
  • Unsigned extension — used with unsigned numbers (copies 0 to the empty bits).

Truncating an integer is when you decrease the number of bytes used to store the integer.
