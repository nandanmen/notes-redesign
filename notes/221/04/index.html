Basic Sorting Algorithms

Beginner Algorithms and Data Structures
221.4

Idea

Data structures that are sorted can greatly improve the performance of other algorithms that wish to act on them. There are two basic sorting algorithms — selection sort and insertion sort.

Selection Sort

Idea

Selection sort sorts the array by recursively/iteratively finding the smallest item in the array and placing it at the front.

Consider the following array:

23
41
33
81
7
19
11
45

Selection sort goes through the array and looks for the smallest number in the entire array — in this case, 7:

23
41
33
81
7
19
11
45

Then places it at the start of the array:

7
23
41
33
81
19
11
45

Essentially this splits the array into a sorted part and an unsorted part. From here, the sorted part will be highlighted blue.

The algorithm continues into the unsorted array, and repeats the entire process. In this case, the algorithm will find 11 and places it at the front of the array:

7
11
23
41
33
81
19
45

This continues until the entire array is sorted.

Implementation

There are two loops required for this implementation — the first loop will swap the smallest item into place, and the inner loop will look for the smallest item.

The following is a C++ implementation:

void selectionSort(int arr[], int size) {
  for (int i = 0; i < size - 1; i++) {
    int smallest = i;
    for (int j = i + 1; j < size; j++) {
      if (arr[j] < arr[smallest]) {
        smallest = j;
      }
    }
    int temp = arr[i];
    arr[i] = arr[smallest];
    arr[smallest] = temp;
  }
}

Let's take the earlier example array and pass it into this function.

i = 0

  • Smallest = 0
  • The inner for loop finds 7, places this index to smallest. So smallest = 4
  • 7 is placed at the front of the list, and the previous front of the list is placed at the location of 7.

After the first iteration, the array now looks like this:

7
41
33
81
23
19
11
45

Performance

The outer loop will run n times. The inner loop makes n - i comparisons per iteration. This means that in total the inner loop runs n(n-1) / 2 times.

Selection sort will perform the same number of comparisons regardless of the original organization of the array (it doesn't matter if the given array was sorted or not).

Since the inner loop runs n(n-1)/2 times, then selection sort can be said to be in O(n2).

Insertion Sort

Idea

Insertion sort works differently than selection sort. It sorts the array by taking the first unsorted item, and placing it in its correct place in the sorted section.

Take the array from before:

23
41
33
81
7
19
11
45

The first item in the array is considered to be sorted, so 23 is highlighted in blue below.

23
41
33
81
7
19
11
45

The algorithm takes 41, the first unsorted item, and compares it to 23. Since it is larger than 23, it is placed after 23 (so essentially there is no swap):

23
41
33
81
7
19
11
45

This process repeats. It takes 33, and compares it to 23. 33 > 23, so it compares it to 41. 33 < 41, so it swaps the positions of 33 and 41:

23
33
41
81
7
19
11
45

Implementation

To implement insertion sort, we have to consider the steps that need to be taken. In each iteration it takes the first unsorted item. So we should assign this to a temporary variable:

for (int i = 1; i < size; i++) {
  int temp = arr[i];
}

Then we should recursively swap this item with its left item, as long as it is larger than its previous. We can do this by shuffling the other items in the list. We would also want to keep track of our current position.

int pos = i;
while(pos > 0 && arr[pos - 1] > temp) {
  arr[pos] = arr[pos - 1];
  pos--
}

Finally we want to place our 33. We now know that it is the correct place because of our previous block of code.

arr[pos] = temp;
