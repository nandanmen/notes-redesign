Abstract Data Types: Queues

Beginner Algorithms and Data Structures
221.8

Idea

Say we have a printer that students can collectively use. It would be fair for the first student to come to be the first to print. The system that allows us to do this is called a queue.

Queue

In a queue items are inserted at the back (end/tail) and removed from the front (head). Thus queues are said to be FIFO (first in, first out) data structures.

At the least, a queue should implement the first two of the following operations:

  • enqueue — inserts an item into the queue
  • dequeue — removes an item from the queue
  • peek — returns the first item in the queue without removing it
  • isEmpty — check if the queue contains any items

Queue Implementation

A queue can be implemented using an array by making the back be the size of the array, and the front originally be index 0.

However, when we try to remove items we will run into some issues. We need to either:

  • Move all the remaining items down, or
  • Increment the index of the front

Neither solution is optimal because one is slow and the other uses up unnecessary memory space.

Solution: Circular Arrays

Using a circular array will allow us to solve these problems. A circular array is when the end of the array "wraps around" to the start of the array.

A queue implemented with a circular array would have the following attributes:

  • front — the front of the queue
  • arrLength — the size of the array
  • arr — a pointer to the array itself
  • num — the number of items in the queue

With these attributes then the back of the queue is:

(front + num) % arrlength

After removal of an item from the queue, the front of the queue is then:

(front + 1) % arrlength
