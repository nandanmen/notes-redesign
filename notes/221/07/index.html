Abstract Data Types

Beginner Algorithms and Data Structures
221.7

Idea

An abstract data type is a collection of data and a set of operations on that data — it describes what data gets collected and what operations to perform, but not how the data is stored or how the operations are performed.

Operations

Mutators

Also known as setters, it changes the contents of an ADT generally by adding or removing data in the collection.

Accessors

Also known as getters, it is able to retrieve data from the ADT and ask questions about the data collection (without manipulating the contents itself).

Example: Stack

A stack ADT only allows items to be inserted and removed at one end. This is referred as the top of the stack.

Access to all other items in the stack is not allowed.

TOP

Stack Behaviour

A stack ADT should support at least the first two of the following operations:

  • push — inserts an item at the top of the stack.
  • pop — remove and return the top item.
  • peek — returns the top item.
  • isEmpty — returns whether the stack contains any items.

Stack Implementation

Stacks can be implemented using a variety of data structures, such as arrays or linked lists. The implementation of a stack, however, requires that:

  • There is a variable that keeps track of the top of the stack,
  • Insertion or deletion time is independent of number of items (runs in O(1)).
