Fibonacci Series

The nth number in the fibonacci series is:

  • 0 if n = 0, and 1 if n = 1
  • fib(n-1) + fib(n-2) for any n > 1

Let's say we would like to write a function that would return the nth fibonacci number. We can simply follow the above formula!

int fib(int n) {
  if (n <= 1) {
    return max(0, n);
  } else {
    return fib(n-1) + fib(n-2);
  }
}

The above function calls itself within the function body. This style of function is called a recursive function.

Recursive Functions

Recursive functions work just like other functions. Each recursive call is placed onto the call stack, and gets removed once the function returns.

Anatomy

Recursive functions consist of two or more cases, and there must be at least one of the following:

  • Base case
  • Recursive case

The base case is the smallest problem where the answer is known (there is no recursive call). There may be more than one base case.

The recursive case is the same problem but with a smaller input.

Stack Overflow

Recall that each recursive call is placed onto the call stack, and will only get removed until the function returns. This means that with a large enough input it is likely that the call stack does not have enough memory to hold all the calls.

A stack overflow is when the call stack runs out of memory.

Tail Recursion

A function is said to be tail recursive if the recursive call is the absolute last thing the function needs to do before returning.

i.e. the function does not need to wait for the other recursive calls to finish.

With tail recursion, compilers can optimize this such that previously used stack space may be reused.

Tail Recursive Factorial

Consider the following function that calculates the factorial of a given int n:

int factorial(int n) {
  if (n <= 0)
    return 1;
  else
    return (n * factorial(n - 1));
}
